Oracle

Reasoning after the benchmark

I care about what models learn, what they only imitate, and what changes once reinforcement learning enters the room. Right now I am working with PEARLS Lab on RL for LLMs and building a corporate simulation environment for long-horizon tool use, finance tasks, and strategy under actual tradeoffs.

Featured paper

Reasoning Through Chess

Lead author on work showing how the composition of fine-tuning data changes downstream reasoning after RL. The project outperformed strong open-source reasoning baselines in chess on a 7B model and focused on how different SFT data regimes shape post-RL behavior, stability, and faithfulness.

Current build

Corporate simulation for multi-turn RL

The goal is to create an environment where a model has to act like it is embedded in a real organization: market dynamics, competition, tool use with costs, data analysis, strategy decisions, and enough friction that long-horizon behavior actually matters.

Now

Environment design with consequence

I am interested in tasks that punish shallow cleverness. That means actions should cost something, state should persist, and the model should have to live with its earlier decisions instead of getting infinite clean retries.

Recently

Fine-tuning data as a hidden steering wheel

The chess work made one thing hard to ignore: the data distribution you use before RL can change what later reasoning looks like, even when headline task performance seems similar from far away.

Questions worth keeping sharp

What kinds of SFT data survive RL without collapsing into performative reasoning?

The qualitative texture of reasoning changes. I want to understand which changes are genuine skill and which are theater.

How do you make tool use costly enough that a model has to strategize instead of spam?

If tools are free, you mostly benchmark patience. If tools have costs, you start to benchmark judgment.

What makes a synthetic corporate task feel enough like real work to matter?

The interesting part is not cosmetic realism. It is persistent incentives, partial information, competition, and awkward tradeoffs.
